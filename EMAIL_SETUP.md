## 📌 功能概述

末末记账本现在支持温暖的邮件推送功能：

## 🚀 快速设置（5分钟）

### 第一步：创建 Resend 账户

1. 访问 [Resend 官网](https://resend.com) （推荐，完全免费）
2. 点击 "Get Started"
3. 用邮箱或 GitHub 账户注册
4. 验证邮箱

### 第二步：获取 API Key

1. 登陆 Resend Dashboard
2. 左侧菜单 → "API Keys"
3. 点击 "Create API Key"
4. 取名（如：momos-expense-tracker）
5. **复制你的 API Key**（形如：re_xxx...）

### 第三步：在 Vercel 设置环境变量

1. 登陆 [Vercel Dashboard](https://vercel.com/dashboard)
2. 选择你的 `momos-expense-tracker` 项目
3. 进入 "Settings" → "Environment Variables"
4. 添加新变量：
   ```
   名称: RESEND_API_KEY
   值: re_xxx...（你复制的 API Key）
   ```
5. 选择环境：Production, Preview, Development（都勾选）
6. 点击 "Save"

### 第四步：重新部署

在 Vercel Dashboard 中：
1. 进入 "Deployments"
2. 找到最新部署，点击右边的三个点
3. 选择 "Redeploy"
4. 等待部署完成（1-2 分钟）

✅ **完成！** 邮件功能现在已激活

## 🐛 故障排查

### 邮件未被发送

**问题：** 点击"邮件"按钮，没有收到邮件

**解决方案：**
1. 检查 Vercel 环境变量是否正确设置
   - Dashboard → Settings → Environment Variables
   - 确认 `RESEND_API_KEY` 存在
   
2. 检查 Vercel 构建日志
   - Deployments → 最新部署 → 查看构建日志
   - 看是否有错误信息

3. 检查浏览器控制台
   - F12 打开开发者工具 → Console
   - 查看是否有 API 错误信息

4. 确认邮箱地址有效
   - 使用真实的邮箱地址注册
   - 检查是否被垃圾邮件过滤

### 邮件发送失败

**错误信息：** "Failed to send email"

**可能原因：**
- API Key 无效或过期
- 邮箱地址格式不正确
- Resend 额度已用完

**解决方案：**
1. 复制新的 API Key 到 Vercel
2. 重新部署应用
3. 查看 Resend Dashboard 的使用额度

### Vercel 部署失败

**错误：** "Cannot find module 'resend'"

**解决方案：**
```bash
# 本地重新安装依赖
npm install resend

# 推送到 GitHub
git add .
git commit -m "添加邮件功能依赖"
git push origin main

# Vercel 自动重新部署
```
